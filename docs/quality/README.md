# 熵管理

Whisper 不维护质量评分、反馈账本、黄金规则清单或泛化熵报告。代码、测试、机械约束和 Git 历史就是长期记录。

`pnpm check` 是阻断性质量门，守护已经确认的类型、测试、构建、架构和安全不变量。熵管理则采用定向人工审查：完成中大型需求、连续完成数项小改动、同类 Bug 再次出现或准备发布时，回看本次涉及模块的职责是否混杂、是否复制已有逻辑、是否留下临时兼容分支，以及代码是否仍容易理解。

发现问题后直接落到离代码最近的位置：行为错误补回归测试，架构越界补 Oxlint 或 Harness 规则，输入问题补类型或 Zod schema，稳定操作方法写入 operations，其他坏味道当场重构。只有真实、重复的问题才升级为新的机械约束。
