# AI Chat 内容与会话交互重设计

日期：2026-07-12

## 背景

当前阅读页的 AI 入口以“框选文字后立即执行阅读动作”为核心：`SelectionMenu` 直接选择解释、定位、概念、背景或例子，`ReadingActionService` 随即创建 thread。右侧面板只展示已经创建的 thread，无法先组织目标、技能和问题，也无法关闭、恢复、删除、重试或引用另一处原文。

本次设计把 AI Chat 改为“先明确本次解读对象，再开始一条稳定会话”。AI 始终保持对整本书的认识，但每条会话只围绕一个固定的核心解读目标。

## 产品术语

- **全书认知**：AI 以何种方式获得整本书的整体背景。
- **解读目标**：本次会话具体针对整本书、某级章节或一段框选内容。
- **技能**：仅用于首次回答的预设处理方式。
- **补充要求**：选择技能后，用户对首次回答增加的可选要求；未选择技能时，同一输入框承载完整问题。
- **引用**：正式会话中附加到单轮追问的一处原文，不改变会话的核心解读目标。
- **草稿**：尚未首次发送、尚未持久化的唯一新会话编辑态。

## 目标

- AI 每次回答都具备全书背景，同时明确区分“全书认知”和“解读目标”。
- 用户可围绕整本书、章节层级或框选内容创建会话。
- 目标、技能、补充要求在首次发送前可组合，发送后形成稳定会话。
- 正式会话之间可通过横向 Tab 切换、关闭并从历史恢复。
- 正式会话中可引用另一处原文追问，也可围绕新选区自动开始新会话。
- 上下文组装按全书认知策略去重，避免把已包含的长章节机械重复塞入模型上下文。

## 非目标

- 首版不支持多技能组合。
- 首版不支持一轮消息引用多处原文。
- 首版不支持会话重命名、历史搜索或历史分组。
- 首版不支持多个未发送草稿。
- 不做跨设备同步。
- 不创建或使用 git worktree。

## 信息模型

### 会话固定信息

首次发送后固定：

- `bookId`
- 全书认知策略 `contextStrategy`
- 核心解读目标 `target`
- 首次技能 `skillType | null`
- 自动生成标题

技能只约束首次回答。后续追问以当轮用户问题为主，技能仅作为历史事实，不再重复注入强制指令。

### 消息信息

用户消息保存：

- 用户可见文本 `content`
- 可选的一处 `reference`

引用保存原文快照及定位信息。发送后输入区引用清空，但消息中的引用可用于展示和“回到原文”。

### 解读目标

```ts
type ReadingTargetType = 'book' | 'chapter' | 'selection';

interface ReadingTarget {
  type: ReadingTargetType;
  chapterId: string | null;
  startPassageId: string | null;
  endPassageId: string | null;
  selectedText: string;
  startOffset: number | null;
  endOffset: number | null;
  breadcrumb: Array<{ chapterId: string; title: string }>;
}
```

- `book`：其余定位字段为空，显示书名或“整本书”。
- `chapter`：`chapterId` 指向选定层级，起止 passage 由章节决定。
- `selection`：保存文本快照、起止 passage 和 passage 内偏移；`chapterId` 指向可完整覆盖选区的最低公共章节，没有公共章节时可为空。
- `breadcrumb` 是创建时的显示快照；定位和上下文组装仍以 ID 为准。

## 全书认知策略

支持：

- `full_book`：完整全书。
- `compressed_book`：全书压缩表示。
- `hybrid`：压缩全书、当前相关章节和关联原文的组合。

规则：

1. 每本书保存默认策略。
2. 新草稿继承该书默认策略。
3. 首次发送前可以只修改本草稿的策略。
4. 正式会话创建后锁定策略。
5. 修改书籍默认策略只影响未来草稿，不改变已有会话。
6. 模型窗口不足导致内部降级时，保存请求策略，并在回答状态中记录有效策略和降级原因；不能静默把会话配置改成降级后的策略。

## 上下文去重原则

解读目标是语义指向，不等于固定追加一整段原文。`ContextAssembler` 分两步工作：

1. 根据 `contextStrategy` 构造全书认知层，并记录其中已覆盖的 passage。
2. 根据目标补足尚未覆盖、但完成任务所必需的细节。

具体规则：

- `full_book` 已完整覆盖目标时，只追加目标类型、章节路径、passage 锚点和短选区，不重复章节全文。
- `compressed_book` 对书或章节目标补充目标摘要与代表性 passage；对框选目标补充框选文本及附近 passage。
- `hybrid` 若已经完整加入目标章节，不再重复；否则补入目标章节或必要片段。
- 长章节目标不默认全文追加，应优先传章节定位、摘要和未覆盖的关键 passage。
- 引用属于当轮消息材料；若引用原文已经完整包含在全书认知层，仍保留短引用文本以消除指代歧义，但不追加整章。

## 章节树前置能力

现有 `Chapter.parentChapterId` 虽已存在，但 Markdown 解析器把所有章节的父 ID 写成 `null`，EPUB 解析器也只生成一级章节。实现面包屑前必须建立可靠章节树：

- Markdown 按 heading depth 使用栈寻找最近的较浅标题作为父章节。
- 每个章节的 `startPassageId` / `endPassageId` 覆盖其自身和所有子章节内容，以便父章节范围成立。
- EPUB 首版保留 spine 一级章节；若源文件没有可用嵌套目录，不伪造父子层级。
- 选区跨多个章节时，计算能够完整覆盖起止 passage 的最低公共祖先；面包屑不展示不能覆盖整个选区的子章节。

## 草稿与首次发送

### 创建

点击右侧 `+` 只打开草稿，不写数据库。草稿默认：

- 目标为整本书。
- 策略继承书籍默认值。
- 未选择技能。
- 输入框为空。
- 没有引用。

同一时间只有一个草稿。

### 自动目标

- 草稿中第一次产生有效选区时，自动把目标切为框选内容。
- 用户手动选择目标层级后，草稿进入手动模式；后续选区不自动覆盖目标。
- 清除目标时退回整本书，并保持手动模式。
- 选区会被捕获为结构化快照，不依赖浏览器原生高亮继续存在。

### 首次发送校验

- 未选择技能：问题文本必填。
- 已选择技能：补充要求可为空。
- 发送成功启动时才创建正式会话和首条用户消息。
- 技能只作用于首次回答。

### 覆盖与关闭

- 正式会话中点击“围绕此处提问”，立即切换到唯一草稿。
- 若已有草稿，直接清空旧目标、技能、输入、引用及草稿策略修改，再以新选区重建草稿，不提示。
- 草稿 Tab 点击 `×` 时直接丢弃，不提示、不进入历史。

## 目标选择器与技能

输入框上方展示目标选择器：

```text
解读目标
[框选内容] [第二节] [第八章] [第三编]    [×]
“因此，所谓自由并不是……”
```

- 框选内容与各级章节按钮互斥单选。
- 章节按钮从最低可用章节向祖先排列。
- 目标分为整本书、章节、框选三类，每类有独立技能列表。
- 技能只能单选。
- 改变目标后重新校验技能；新列表仍有同一技能则保留，否则清除并轻提示。
- 不自动映射相似技能。

输入框占位文案：

- 未选技能：`你想了解什么？`
- 已选技能：`可补充具体要求，也可以直接发送`

## 正文选区菜单

保留靠近选区的精简菜单：

- 当前为草稿：`设为解读目标`
- 当前为正式会话：`围绕此处提问`、`引用到当前会话`

技能、章节层级和输入全部留在右侧面板，不在浮动菜单重复。

## 正式会话中的引用

点击“引用到当前会话”后，在 Composer 上方显示附件：

```text
[引用：第五章 > 第二节]                       [×]
“这里是一段新框选的文字……”
```

规则：

- 每轮最多一处引用，新引用替换尚未发送的旧引用。
- 引用不改变会话核心目标。
- 有引用时必须输入追问，不能单独发送引用。
- 引用不混入可编辑文本。
- 消息持久化引用快照，并提供“回到原文”。

## Tab 与历史

顶部采用横向滚动 Tab：

```text
[会话 A ×] [会话 B ×] [会话 C ×] … [+] [历史]
```

- 不限制打开数量。
- 正式会话 `×` 仅关闭视图，不删除数据库记录。
- 正在生成时允许关闭，生成继续在后台运行。
- 历史列表统一按 `updatedAt DESC`，生成中的会话置顶，不分组、不搜索。
- 点击历史记录后恢复为 Tab；若已打开则只激活。
- 删除只在历史列表提供，并需要二次确认。
- 删除活动会话后激活相邻 Tab；没有正式 Tab 时显示草稿或空态。
- 打开历史会话不改变正文滚动位置。

为了区分“所有历史”和“当前打开”，前端维护每本书的 `openThreadIds`。首版可持久化到 renderer 的 `localStorage`；数据库只保存 thread 与消息，不把纯视图状态混入领域表。无缓存时默认只打开最近活动的会话。

## 标题

不调用 AI，首次发送时按规则生成：

- 有技能：`目标摘要 · 技能`
- 无技能：`目标摘要 · 首问摘要`

框选目标取选区开头，章节目标取章节标题，书目标显示“全书”。标题限制约 18 个中文字符并省略超出部分。首版不支持手动重命名。

## 回到原文

会话头部和带引用的用户消息提供“回到原文”：

- 章节目标滚动到 `startPassageId`。
- 框选或引用先按 passage ID 和偏移恢复并临时高亮。
- 偏移失效时，在目标 passage 中查找文本快照。
- 仍无法精确匹配时滚动到起始 passage 或章节，并提示只能恢复到附近原文。
- 打开历史会话本身不自动滚动。

## 失败与重试

- 失败时保留会话、首问、目标快照和空的 assistant 占位消息。
- 点击重试复用同一 assistant 消息 ID，清空旧的部分内容并重新生成。
- 不新增重复用户消息或 assistant 消息。
- 重试使用会话锁定的策略与目标。
- 后续追问失败也按相同规则重试最后一条失败回答。

## 数据持久化

### `reading_threads`

新增或替换为以下语义字段：

- `target_type TEXT NOT NULL`
- `target_chapter_id TEXT`
- `target_start_passage_id TEXT`
- `target_end_passage_id TEXT`
- `target_selected_text TEXT NOT NULL DEFAULT ''`
- `target_start_offset INTEGER`
- `target_end_offset INTEGER`
- `target_breadcrumb_json TEXT NOT NULL DEFAULT '[]'`
- `skill_type TEXT`
- `context_strategy TEXT NOT NULL`
- `last_error TEXT`

旧字段 `chapter_id`、`passage_id`、`action_type`、`selected_text` 首版迁移时保留兼容读取；新写入只使用新字段。旧会话映射为 `selection` 目标和原动作技能。

### `thread_messages`

新增：

- `reference_json TEXT`
- `status TEXT NOT NULL DEFAULT 'complete'`，取值为 `streaming | complete | failed`
- `error TEXT`

必须使用启动时幂等列迁移，兼容现有 SQLite 数据。

## IPC 与服务边界

新增或重构接口：

```ts
createConversation(input: CreateConversationInput): Promise<ThreadWithMessages>
followUp(input: FollowUpInput): Promise<ThreadWithMessages>
retry(input: { threadId: string; messageId: string }): Promise<ThreadWithMessages>
deleteThread(input: { threadId: string }): Promise<void>
```

- renderer 草稿不通过 IPC 持久化。
- 主进程验证目标、技能和问题组合，不能只依赖前端校验。
- 删除 thread 与其 messages 使用事务；若书籍 `active_thread_id` 指向被删 thread，同时清空。
- 流式事件继续按 thread/message ID 更新，即使对应 Tab 已关闭也要更新历史状态。

## 状态与并发

- 每个 thread 独立拥有 `streaming | ready | failed` 状态。
- renderer 不使用单一全局 `streamError`，错误绑定到具体 thread/message。
- 同一 thread 正在生成时禁止再次发送；不同 thread 可继续独立展示后台状态。
- 关闭 Tab 不取消生成。
- 页面卸载不会主动取消主进程生成。

## 验收标准

1. 点击 `+` 不产生数据库 thread，首次发送才产生。
2. 草稿首次框选自动成为目标，手动选章节后新选区不覆盖目标。
3. Markdown 多级标题能生成正确面包屑；跨兄弟章节选区只展示公共父章节。
4. 三类目标展示各自技能，目标变化会清除不适用技能。
5. 有技能时允许空补充要求，无技能时禁止空发送。
6. `full_book` 不重复追加章节全文；压缩与混合策略能补足目标必要细节。
7. 正式会话中新选区可分别触发新草稿或单轮引用。
8. 一轮只有一个引用且必须附带问题，发送后消息可回到原文。
9. 正在生成的 Tab 可关闭且后台继续，历史状态持续更新。
10. 历史按更新时间倒序、生成中置顶，可恢复、关闭和删除。
11. 打开历史不自动滚动，点击“回到原文”才定位并高亮。
12. 重试覆盖失败回答，不创建重复消息。
13. 旧数据库可迁移，旧会话仍可打开。

## 风险与取舍

- DOM Selection 到 passage/offset 的映射是交互可靠性的关键，必须单独测试，不能只保存 `window.getSelection().toString()`。
- 章节摘要目前大多为空；压缩策略首版仍使用 passage 采样，后续可接入预处理摘要。
- Markdown 章节范围语义改变后，需要确保既有解析测试覆盖父章节包含子章节的范围。
- `localStorage` 中的打开 Tab 列表属于视图恢复能力，清缓存不会丢会话，只会丢当前打开布局。
