# 阅读器层级目录设计

日期：2026-07-15

## 背景

阅读器左侧目录当前将 `BookDocument.chapters` 线性渲染为相同样式的链接。书籍解析和存储已经通过 `Chapter.parentChapterId`、`level` 与 `order` 保留章节层级，但界面没有表达父子关系。以《语言的魔力》为例，Part 与下属章节被压平成同一级，读者无法通过目录理解全书结构。

不同书籍使用 Part、篇、章、节或无编号标题，标题深度和起始级别也不一致。目录不能通过标题文字猜测章节语义，必须使用通用结构规则。

## 目标

- 把左侧目录改为安静、可折叠的层级树，让读者快速理解章节从属关系。
- 同时支持全书结构浏览和当前位置导航。
- 对不同标题名称、起始级别和嵌套深度采用一致规则。
- 保持原书为阅读页视觉主角，目录层级清晰但不过度抢夺注意力。

## 非目标

- 不修改 Markdown 或 EPUB 的解析规则。
- 不修改数据库 schema、IPC 契约、preload API 或主进程服务。
- 不根据 `Part`、章、节、前言等标题文本推断语义或创建虚拟分组。
- 不在本次增加目录搜索、阅读进度、书签或用户自定义层级上限。
- 不把第五层及以后章节从书籍数据或正文中删除。

## 结构模型

Renderer 根据 `parentChapterId` 和 `order` 将线性章节列表构建为有序树。目录层级使用树的相对深度，不直接使用 Markdown 标题数字：无父节点的章节是目录第 1 层，其子节点是第 2 层，以此类推。因此，即使一本书从 `##` 开始，其根章节仍占用目录第 1 层。

目录最多展示四层。第 5 层及更深章节继续保留在 `BookDocument` 和正文中，但不生成目录节点。任意深层章节都映射到最近的可见祖先；当阅读位置落在被省略的章节内时，该祖先承担目录高亮。

目录不理解章节名称。CIP、译序、致谢、前言等没有子节点的顶层章节作为普通顶层叶节点；Part 或其他有子节点的顶层章节自然成为可展开父节点。是否出现折叠控件只由节点是否存在可见子节点决定。

异常结构采用保守降级：

- 找不到父节点的章节作为根节点显示；
- 检测到父子循环时停止沿该路径递归，避免无限渲染；
- 没有章节时保持阅读界面可用，不生成伪造目录项；
- 没有有效正文锚点的节点可以展开，但标题不触发无效定位。

## 视觉表达

目录采用安静树形，不使用卡片式章节分组。

- 每深入一层增加固定缩进，最多四层，因此不需要无限压缩标题宽度。
- 每个展开的父节点显示一条连续竖线，覆盖其全部可见后代并在最后一个后代处结束。
- 多层父节点同时展开时显示并列祖先线，使读者能够沿线追溯当前节点的完整从属关系。
- 叶节点不显示折叠箭头；父节点使用轻量箭头表达展开状态。
- 当前节点使用低亮琥珀背景和左侧强调线；祖先节点不使用同等强度高亮。
- hover、键盘焦点与当前状态必须可以区分；键盘焦点使用清晰的独立轮廓。

目录继续使用现有深胡桃木结构色和低对比度次级文字，遵守视觉系统中“原书是第一视觉焦点”的约束。

## 展开与导航

首次进入书籍时，当前阅读位置的祖先链自动展开，其余父节点默认折叠。正文滚动到新的分支时，目录自动更新当前节点并展开新位置所需的祖先。自动跟随不关闭用户手动展开的其他分支。

父节点的箭头和标题是独立操作目标：

- 点击箭头只展开或折叠直接子树；
- 点击标题只将正文滚动到对应章节位置；
- 点击标题不改变其他分支或当前分支的展开状态。

用户手动折叠优先于当前分支的自动展开。用户折叠包含当前章节的分支后，只要正文仍在该子树内，目录保持折叠，并将被折叠的最近可见父节点标记为当前位置。正文滚动到另一个分支时，系统再展开新位置所需的祖先；此前由用户折叠的分支保持折叠。

目录定位完成后，正文位置与当前高亮通过同一套滚动跟随逻辑收敛，不维护第二份正文当前位置。目录组件可以维护展开与显式折叠集合，以区分用户操作和自动展开。

## Renderer 边界与数据流

本次在 renderer 内增加两个职责单一的单元：

1. 纯目录模型：输入线性 `Chapter[]`，输出最多四层的有序目录树，以及“任意章节 ID 到最近可见节点 ID”的映射。该单元负责孤儿、循环、深度上限和稳定顺序。
2. 目录组件：递归渲染可见树，维护手动展开集合，接收当前章节 ID，并发出展开切换和正文定位意图。

正文阅读位置根据阅读基准线附近的 passage 确定。passage 已关联其所属章节；读取当前 passage 的 `chapterId` 后，通过目录模型映射成可见节点。实现可以给正文节点补充必要的 `data-*` 元数据，但不得把目录状态写回主进程或数据库。

展开状态只在当前 ReaderPage 生命周期内保留，本次不跨应用重启持久化。

## 无障碍与动效

- 目录保留带名称的 `nav` landmark。
- 层级结构使用语义列表；折叠按钮暴露 `aria-expanded`，并具有包含章节标题的可访问名称。
- 标题定位和折叠必须能够分别通过键盘操作。
- 当前目录项使用 `aria-current="location"` 表达当前位置，不能只依赖颜色。
- 展开动效应克制，并遵守 `prefers-reduced-motion`；没有动效也不得影响结构理解。

## 验收条件

- 两层、四层和超过四层的章节数据都能生成稳定、正确的目录树。
- 目录深度按父子关系计算，不受 Markdown 起始标题级别影响。
- 第 5 层及以后不显示在目录，但正文仍然存在，并能高亮最近的可见祖先。
- 展开的父节点竖线贯穿全部可见后代；嵌套父节点显示并列祖先线。
- 顶层叶节点和顶层父节点可以自然混排，不依赖标题命名。
- 箭头只控制折叠，标题只控制正文定位。
- 正文滚动会更新当前目录项并展开新分支的必要祖先，不会关闭用户手动展开的其他分支。
- 用户折叠当前分支后目录不会自行弹开，被折叠的最近可见父节点会承接当前位置标记。
- 孤儿节点、循环关系、空目录和无有效锚点不会导致崩溃或无限递归。
- 目录支持键盘操作，展开状态和当前位置具有正确的无障碍语义。
- 相关 renderer 单元与交互测试通过，完整 `pnpm check` 通过。

## 测试策略

- 目录模型单元测试覆盖正常嵌套、乱序输入、相对深度、孤儿节点、循环、四层截断和深层映射。
- 目录组件测试覆盖递归结构、折叠按钮、标题定位、当前状态、新分支祖先自动展开、当前分支手动折叠优先和手动展开状态保留。
- ReaderPage 回归测试覆盖 passage 到当前章节的跟随、深层章节回退，以及空目录场景。
- 样式或结构测试验证连续祖先线所需的 DOM 层级，人工验收确认常见窗口宽度下四层标题仍可阅读。
