# AI 右栏任务式 Composer 重设计

日期：2026-07-17

## 背景

当前右栏底部把解读目标、章节层级、技能、全书认知、自由输入和发送按钮全部常驻铺开。在约 360px 宽的阅读侧栏中，这些不同层级的操作互相争夺空间，也把首轮解读误呈现成以文本输入为中心的普通聊天。

实际使用中，用户往往只需指定“读哪里”和“怎样解读”即可开始；文字问题只是可选补充。会话建立后，解读目标和方式已经固定，后续只需围绕既有回答追问。

本设计保留顶部多 Tab 和中间聊天区，只重构底部 Composer、正式会话任务摘要、正文选区入口和全书认知入口。本文取代 `docs/specs/completed/2026-07-12-ai-chat-redesign.md` 中与草稿 Composer、正式会话引用和全书认知草稿覆盖有关的界面规则；未明确修改的多 Tab、历史、流式回答、删除和重试行为继续有效。

## 产品术语

- **解读任务**：首次发送前，由解读目标、解读方式和可选补充提问组成的任务配置。
- **解读目标**：AI 本次具体解读的整本书、某级章节或一段框选内容。
- **解读方式**：仅约束首次回答的单选处理方式；取代用户界面中的“技能”一词。
- **补充提问**：首轮可选的额外要求，用于收窄或强调解读重点。
- **追问**：正式会话中的后续文字消息，不改变会话的解读目标或解读方式。
- **全书认知**：AI 获得整本书背景的策略，是当前书的新会话默认值。

内部领域字段 `skillType` 和 `ReadingSkillType` 暂不重命名。本次只改变用户可见术语，避免无收益的数据迁移和跨进程契约改名。

## 目标

- 把首轮交互从普通聊天输入改为结构化解读任务。
- 在有限侧栏空间内同时露出解读目标、当前目标适用的全部解读方式和可选补充提问。
- 允许只凭解读目标和解读方式开始，不强迫用户输入问题。
- 会话建立后固化任务身份，后续只支持文字追问。
- 把低频的全书认知策略移出日常 Composer，按书管理。
- 保持现有多 Tab、中间聊天区、历史和流式生成能力。

## 非目标

- 不支持一次选择多种解读方式。
- 不支持正式会话中修改解读目标或解读方式。
- 不支持正式会话追问附加新原文引用。
- 不重新设计顶部 Tab、中间消息样式或历史列表。
- 不重命名现有数据库字段或跨进程类型中的 `skillType`。
- 不改变模型、API key、上下文窗口等应用级设置。

## 两阶段会话模型

右栏会话分为两个互斥阶段。

### 首轮：建立解读任务

```text
解读目标   [整本书                                      ⌄]
解读方式   [当前目标适用的 3–4 种方式，单行等分]
补充提问   [还有特别想了解的吗？                    可选]
                                               [开始解读]
```

最低有效条件：

- 解读目标有效。
- 已单选一种解读方式。
- 补充提问可以为空。

### 后续：围绕固定任务追问

```text
框选内容 ↗ · 白话解释                            只读

                  对话消息

[继续追问……                                      ↑]
```

首次创建成功后，目标和解读方式固化为会话身份。后续 Composer 只接受文字追问，不显示目标、方式、附件或引用入口。

改变目标或方式必须创建新会话。

## 首轮紧凑任务区

### 尺寸与布局

- 以约 360px 可用宽度设计。
- 三个字段均使用约 30px 行高。
- 标签列固定约 66px，内容列占用剩余宽度。
- 字段间距约 7px。
- 默认未展开时，整个任务区目标高度约 165px。
- 不允许横向滚动；当前目标适用的 3–4 种解读方式不得换行或隐藏。

精确像素可以在实现中随现有设计 token 微调，但必须保持单行方式选择和紧凑垂直占用。

### 解读目标

默认只显示当前值和展开箭头，不常驻铺开章节层级。

无有效正文选区时：

- 新会话默认目标为整本书。

有有效正文选区时：

- 新选区自动成为草稿目标。
- 单行摘要显示“框选内容”和截断后的原文，不铺开整段文本。
- 草稿仍保存完整结构化选区快照。

点击目标后打开按需菜单：

- 整本书。
- 当前小节。
- 当前章节及可用祖先层级。
- 框选内容；仅在存在有效选区快照时出现。

章节层级沿用现有 `ReadingTarget` 表达，不新增“本节”或“本章”目标类型。

### 解读方式

- 用户界面统一使用“解读方式”，不显示“技能”。
- 无默认值，用户必须主动选择。
- 单选。
- 方式沿用现有按目标分类：
  - 整本书：总结全书、提炼框架、评价全书。
  - 章节：概括本章、章节作用、梳理论证。
  - 框选内容：白话解释、解释概念、补充背景、举例类比。
- 当前目标适用的全部方式在一行内等分展示；三项使用三等分，四项使用四等分。
- 选中项使用明确的整体选中态；未选项保持低视觉权重。
- 切换解读目标后，若当前方式不适用于新目标，则清除选择并提示重新选择；不自动映射相似方式。

### 补充提问

- 始终露出，但明确标记“可选”。
- 默认单行，聚焦后可以在原位扩展到 2–3 行。
- 失焦且内容为空时恢复单行。
- 内容非空时保持可读，不因失焦截断用户已输入内容。
- 文案作为首次用户消息的可见内容；为空时仍可创建会话。

### 开始解读

- 未选择有效解读方式时禁用。
- 禁用时可显示轻量提示“请选择解读方式”。
- 有效后按钮文案为“开始解读”，不使用泛化的“发送”。
- 创建失败时保留全部草稿内容，显示错误并允许重试。
- 收到首次流式 `started` 事件后沿用现有行为，立即打开正式会话并展示生成过程。

## 正式会话

### 常驻任务摘要

顶部 Tab 下方常驻一条约 28px 高的只读摘要：

```text
框选内容 ↗ · 白话解释                            只读
```

- 摘要显示目标简名和解读方式。
- 不使用表单控件，不暗示可编辑。
- 框选或章节目标可点击回到原文并恢复定位提示。
- 整本书目标不显示定位动作。
- 目标无法定位时保留文本快照与会话内容，并提示“无法定位到原文”；不得导致会话不可用。
- 不再单独展示 `全书认知：hybrid` 等技术状态文案。

### 追问区

- 底部只显示约 46px 的文字追问 Composer。
- 占位文案为“继续追问……”。
- 空追问不能发送。
- 不显示解读目标、解读方式、正文引用或附件入口。
- 后续模型上下文仍包含既有会话历史和固定任务信息，但不再次把解读方式作为新的强制动作注入。

## 正文选区行为

### 草稿阶段

- 有效新选区自动替换草稿中的解读目标。
- 选区浮层提供“设为解读目标”仅在自动带入未发生或需要显式确认的场景使用；实现时应避免同一选区出现重复动作。

### 正式会话阶段

- 新选区浮层只提供“新建解读”。
- 点击后打开唯一新会话草稿，并自动带入该选区。
- 不提供“引用到当前会话”或“插入追问”。
- 当前正式会话保持打开，用户可以通过 Tab 返回。

现有 `MessageReference` 历史数据继续可读、可展示和定位；本次只移除创建新引用的界面路径，不要求数据库迁移或删除旧数据。

## 全书认知设置

### 入口与范围

- 右栏顶部提供书籍级设置入口。
- 入口打开紧凑 Popover，不占用 Composer 垂直空间。
- Popover 只承载当前书的全书认知策略，不混入模型、API key 或上下文窗口等应用设置。

### 策略

- 完整全书 `full_book`。
- 压缩全书 `compressed_book`。
- 混合 `hybrid`。

每项在 Popover 中提供简短解释。选择后立即保存，无需额外“保存”按钮。

### 生效语义

- 设置按书保存为新会话默认值。
- 创建会话时将当前书默认策略复制为会话的 `contextStrategy` 快照。
- 修改只影响之后创建的新会话。
- 既有会话不随默认值变化。
- 保存失败时恢复原选项并提示错误；不得修改草稿或既有会话的策略。

应用级 `defaultContextStrategy` 仍可作为新导入书籍或没有书籍覆盖值时的默认来源，但不在右栏 Popover 中编辑。

## 数据与兼容性

现有领域模型可以继续承载新交互：

- `target`：首轮选择并在创建时固化。
- `skillType`：首轮必选，用户界面称“解读方式”。
- `prompt`：可为空，对应补充提问。
- `contextStrategy`：从当前书默认值复制到会话。
- `reference`：旧消息继续兼容，新追问不再创建。

草稿校验从“未选技能时问题必填”改为：

```text
target 有效 AND skillType 非空
```

`prompt` 不再参与是否允许首次创建的判断。

本次不需要 SQLite schema 迁移。若实现中发现现有书籍记录无法保存独立的全书认知策略，应优先复用现有 `books.setContextStrategy` 契约；不得把书籍级覆盖退化为 renderer 临时状态。

## 组件职责建议

- `DraftComposer`：编排首轮任务表单和提交校验。
- `TargetPicker`：改为单行摘要与按需菜单，只负责目标选择。
- 新的 `InterpretationMethodPicker` 或等价小组件：负责当前目标适用方式的单行等分单选。
- `ThreadChat`：展示只读任务摘要和纯追问 Composer。
- `RightAiPanel`：承载书籍级全书认知入口，不把设置状态混入单个会话 Composer。
- 选区菜单：根据当前视图只派发“更新草稿目标”或“新建解读”。

组件命名可在实施计划中结合现有文件边界调整；职责边界不得重新把目标、方式、书籍设置和追问塞回同一组件。

## 错误与边界情况

- **选区快照仍在但定位失败**：允许创建和阅读会话；定位操作给出非阻塞提示。
- **目标变化导致方式不适用**：清除方式并禁用“开始解读”。
- **创建请求失败**：保留草稿三项内容，不创建残缺的正式会话视图。
- **流式生成失败**：沿用现有失败消息与重试机制。
- **全书认知保存失败**：恢复原值并提示；新会话继续使用最后成功保存的值。
- **生成中关闭或切换 Tab**：生成继续在后台进行，沿用现有语义。
- **旧会话含消息引用**：继续正确渲染和定位，不提供新增入口。

## 验收条件

### 首轮任务

- 约 360px 宽时，三个字段无横向滚动，当前目标适用的 3–4 种解读方式保持单行。
- 无选区的新会话默认目标为整本书。
- 有效选区自动成为草稿目标并显示单行摘要。
- 解读方式无默认值且只能单选。
- 未选解读方式时不能开始。
- 已选目标和方式、补充提问为空时可以创建会话。
- 补充提问聚焦时扩展，空内容失焦后收起。

### 正式会话

- 创建后首轮表单消失，显示只读任务摘要和追问区。
- 摘要滚动聊天时保持可见。
- 可定位目标能够回到原文；定位失败不会破坏会话。
- 后续只能发送非空文字追问。
- 正式会话不显示修改目标、解读方式或新增引用的入口。

### 选区与新会话

- 正式会话中新选区只提供“新建解读”。
- 新建解读自动带入该选区，原会话仍保留在 Tab 中。
- 不再能从 UI 创建正式会话消息引用。

### 全书认知

- 右栏顶部可查看并修改当前书策略。
- 修改成功只影响随后创建的会话。
- 既有会话的策略快照保持不变。
- 保存失败会恢复原选择并给出提示。

### 回归

- 多 Tab 打开、切换和关闭行为不变。
- 历史恢复、删除和重试行为不变。
- 首次和后续回答继续流式可见。
- 旧引用消息继续兼容展示。
- `pnpm check` 通过。

## 人工验收

在 `docs/MANUAL_TESTING.md` 增加或更新以下关键旅程：

1. 无选区创建整本书解读，不填写补充提问。
2. 框选原文后创建解读，并从正式会话摘要回到原文。
3. 正式会话中框选另一段，创建新解读并在两个 Tab 间切换。
4. 修改当前书全书认知，验证只影响新会话。
5. 在窄侧栏下检查首轮三行布局、方式单行和补充提问展开。
