# 人工验收清单

自动检查通过后，只执行与本次改动相关的项目；准备发布时执行完整清单。记录操作系统、应用版本、检查日期、结果和异常，不以本清单代替单元或集成测试。

## 启动与设置

- [ ] `pnpm dev` 能启动应用，主窗口无白屏，开发者控制台无未说明错误。
- [ ] 保存 OpenAI-compatible 设置后重新打开设置，非敏感字段正确恢复。
- [ ] API key 不出现在终端和主进程日志中。
- [ ] 测试连接成功与失败都有清晰、可恢复的反馈。

## 书籍导入与阅读

- [ ] 导入 Markdown 后书名、章节层级和正文正确，重复或异常文件有明确反馈。
- [ ] 导入 EPUB 后书名、章节层级和正文正确。
- [ ] 打开书籍、切换章节和返回书库正常。
- [ ] 框选跨行原文后，选区菜单位置合理且目标文本准确。

## AI 会话

- [ ] 不框选正文新建会话时，解读目标默认为整本书，解读方式无默认值且“开始解读”禁用。
- [ ] 选择一种整本书解读方式、不填写补充提问时，可以开始并立即看到流式回答。
- [ ] 框选正文并点击“新建解读”后，目标自动变为框选内容，四种解读方式保持单行显示。
- [ ] 约 360px 宽的右栏中，首轮三行任务无横向滚动；补充提问聚焦后扩展，空内容失焦后收起。
- [ ] 首次发送后立即出现用户消息与“模型思考中”，回答逐块显示。
- [ ] 会话建立后任务摘要常驻且只读，底部只支持非空文字追问；可定位目标能回到原文。
- [ ] 正式会话中框选另一段并新建解读时，原 Tab 保留，新草稿带入新选区。
- [ ] 修改当前书的全书认知后，新会话使用新策略，既有会话保持创建时的策略。
- [ ] 旧版本产生的消息引用继续正确展示并可定位，但界面不再提供新增引用入口。
- [ ] 切换或关闭 Tab 时后台流不丢失，也不会抢占当前视图。
- [ ] 模型失败后错误状态清晰，重试不会产生异常重复消息。
- [ ] 历史会话可恢复、删除，当前会话选择符合预期。

## 持久化与恢复

- [ ] 重启应用后书库、设置、书籍内容、历史消息和上次活跃会话正确恢复。
- [ ] 使用修改前版本产生的数据启动后，符合本次 schema 兼容策略且无静默数据丢失。
